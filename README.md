# 喷涂大房子 · Paint House

一个多人在线的喷涂游戏：所有玩家在一个 3D 房子里喷涂地板。
**只能在自己颜色的地板上行走，颜料可以互相覆盖**，倒计时结束按颜色面积排行。

## 玩法

- WASD / 方向键 移动
- 鼠标移动 控制朝向（角色面向鼠标）
- 左键 / 空格 喷涂
- 60 秒倒计时
- 颜料覆盖面积最大的玩家获胜

## 启动

```bash
npm install
npm start
```

服务器跑在 `http://localhost:3000`。在两个或多个浏览器窗口（或多台机器）同时打开此地址即可联机。任意一名玩家点击"开始游戏"开始一局。

## 项目结构

| 路径 | 说明 |
|---|---|
| `server.js` | Node.js + WebSocket 服务，权威游戏状态、面积统计、倒计时 |
| `public/index.html` | HTML 入口，使用 importmap 引入 Three.js |
| `public/js/main.js` | 主程序：Three.js 初始化、相机、输入、网络、UI |
| `public/js/scene.js` | 程序化生成的现代 loft 风格房子内部（墙、家具、灯光） |
| `public/js/painter.js` | 地板喷涂：CanvasTexture + 颜色采样网格（用于行走限制） |
| `public/js/network.js` | 简易 WebSocket 客户端（事件分发） |

## 技术要点

- **房子模型**：程序化生成（开放式 loft：客厅、餐厅、厨房、卧室）。结构里预留了
  GLTF 加载入口，把 `.glb` 文件丢进 `public/models/` 替换 `scene.js` 的程序化几何即可。
- **喷涂**：地板用 1024×1024 的 CanvasTexture 实时绘制软边圆形，同时维护一份
  256×256 的 owner-id 网格用来判定行走权限。
- **行走限制**：客户端预测 + 服务端权威双重校验。客户端在
  `canStandAt(x, z)` 里采样 owner，服务端在收到 `move` 时同样采样，若是敌方颜色
  就回包 `snapBack` 把玩家拉回。
- **多人同步**：所有喷涂事件由服务端广播，每个客户端独立把同样的笔触画到本地
  Canvas 上，保证视觉一致。本地玩家的喷涂走"乐观应用 + 跳过自己回包"避免抖动。
- **面积统计**：服务端的 256×256 网格随每次喷涂累计，每 0.5 秒广播一次实时分数；
  局终用同一份网格计算最终排名。

## 颜色席位

最多支持 8 位玩家：红、青、黄、绿、紫、橙、薄荷、品红，按连接顺序分配。
